# /os-task — 任务管理

你需要帮助用户管理 AI Team OS 中的团队任务。

## 用法

- `/os-task` — 列出所有团队的任务
- `/os-task run <描述>` — 创建并执行一个任务
- `/os-task <task_id>` — 查看任务详情

## 操作流程

### 无参数：列出任务

1. 调用 `team_list` 获取所有团队
2. 对每个团队调用 `task_list` 获取任务列表
3. 按状态分组展示：进行中 → 待处理 → 已完成

### run 模式：创建并执行任务

1. 向用户确认：
   - 任务描述（如果用户已提供则直接使用）
   - 目标团队（如只有一个团队则自动选择）
2. 调用 `task_run` 执行任务
3. 展示执行结果，包含各 Agent 的输出

### 查看模式：任务详情

1. 调用 `task_get` 获取任务信息
2. 展示任务详细信息：状态、分配、时间、结果

## 输出格式

### 任务列表
```
## 团队任务

### dev-team
| 状态 | 任务ID | 标题 | 分配给 | 创建时间 |
|------|--------|------|--------|----------|
| 进行中 | t-001 | 实现登录模块 | backend-dev | 2026-03-13 |
```

### 任务执行结果
```
## 任务执行完成

任务ID: t-001
状态: 已完成
耗时: 45.2秒

### 结果
<汇总结果>

### Agent输出
**backend-dev**: <输出内容>
**frontend-dev**: <输出内容>
```

## 注意

- 所有输出使用中文
- 任务状态使用中文标记: pending=待处理, running=进行中, completed=已完成, failed=失败
